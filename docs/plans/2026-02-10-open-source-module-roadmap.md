# 开源功能模块路线图（草案）

日期：2026-02-10

## 已锁定的优先项

以下两项已正式记录为第一优先级缺口：

1. 图片与图注模块
   - 理由：`docs/supported-formats.md` 中图片能力仍为占位，报告场景对图片是刚需。
   - 当前缺口：formatter 流程里缺少清晰的图片 AST 到 DOCX 渲染链路。
   - 目标结果：支持标准 Markdown 图片语法导出，并可控制尺寸、对齐、图注与编号。

2. 参考文献源管理模块
   - 理由：当前引用流程只提取 `[n]` 并生成占位文献，真实文献信息仍需手工补齐。
   - 当前缺口：缺少来源导入（BibTeX/手工条目）、样式输出（GB/T、APA、IEEE）与复用机制。
   - 目标结果：统一管理文献源，自动映射文内引用并生成格式化参考文献列表。

## 范围锁定

按当前决策，仅做以下两项，不扩展其他功能模块：

1. 图片与图注模块
2. 参考文献源管理模块

其余候选模块全部延期，后续如需恢复再单独立项。

## 建议实施顺序

1. 图片与图注模块
2. 参考文献源管理模块
